# Support Portal Migration — Alignment sync within Flamingo

Tue, 14 Apr 26 · kristina.lamza@tourlane.com, markus.klemann-ext@tourlane.com, PR-20 HQ-2-Spain (10) [Camera, Google Meet, Mic, TV, Tablet]

### Migration Scope & Timeline

- Decision: Proceeding with Option 4A (authorized section migration to CFA) only
  - Estimated 2-3 sprints with Markus full-time
  - Option 4B (full migration) would require 5-10 additional sprints - not feasible
- Migration candidates behind login:
  - Finance components
  - Trip visualization (light version)
  - Emergency contacts
  - Invoice section (Salesforce data)
  - Admin view components

### Technical Architecture & Shared Components

- CFA integration approach confirmed feasible
  - Support portal keeps current URL/login structure
  - Backend functions migrate to CFA while frontend remains
  - Shared capabilities won’t conflict with new user portal design
- Overlapping components between portals:
  - Login system (both portals need authentication)
  - Trip visualization (different complexity levels)
  - Chatbot/AI features (shared across platforms)
  - FAQ content management

### Design & Planning Dependencies

- Major blockers identified:
  - No UI designs exist yet for new user portal migration components
  - Vision prototype from Steff needs official review/approval
  - Foxes team status on user portal development unclear
- Required before migration starts:
  - Joint workshop with Foxes team (half-day session)
  - Design capacity allocation (Shubham availability uncertain)
  - Technical milestone breakdown with both teams
- Timeline risk: May start date questionable due to alignment needs

### Next Steps

- Kristina: Discuss vision prototype status with Steff for broader team sharing
- Kristina: Clarify design resource availability with Shubham and Lena today
- Kristina: Document shared capabilities list for technical review
- Team: Organize joint workshop with Foxes team next week
- Decision needed: Completeness check coverage during Markus migration focus
- Update estimate to 4 sprints + additional product/design overhead
