# Summary from Granola:

### Support Portal Migration Overview

- Markus wrote comprehensive migration guide: [https://tourlane.atlassian.net/wiki/spaces/TPD/pages/4651155469/Support+Portal+to+CFA+Migration+Roadmap](https://tourlane.atlassian.net/wiki/spaces/TPD/pages/4651155469/Support+Portal+to+CFA+Migration+Roadmap)
- Current portal built as monorepo with Next.js and TypeScript
- Uses Strapi CMS for FAQ content management
  - Supports 3 languages (German, French, English)
  - Vector database for AI-powered search
  - PostgreSQL backend via Supabase
- Migration target: integrate into CFA project using React and new UI framework

### Phase 1: Authenticated Customer Area

- Scope: migrate logged-in user functionality to user portal
  - Trip visualization (already live since last Monday)
  - Invoice section (fetches from Salesforce)
  - Emergency contact information
  - Same components used across guest/admin views
- Effort estimate: 2-3 sprints (Markus full-time focus)
- Risk level: Medium-Low
- Main challenge: converting Next.js components to React with new design system

### Phase 2: AI Support Form & Chat

- Core functionality migration
  - General support form with AI responses
  - Modification chat for trip changes
  - Vector store integration for FAQ matching
  - Salesforce case creation workflow
- Current metrics: 14,000 customer requests processed
- Effort estimate: High complexity, high risk
  - Heavy AI logic tightly coupled to Next.js
  - Vector database dependencies
  - Sensitive customer interaction flows
- Requires complete rewrite of AI response system

### Phase 3: Admin Dashboard

- Employee support tools
  - Customer request history view
  - Trip data access for support agents
  - Analytics dashboard (replacing current Google Sheets)
  - Support request tracking and management
- New admin view launching next week
- Requires separate CFA sub-application for employee access
- Lower priority than customer-facing features
