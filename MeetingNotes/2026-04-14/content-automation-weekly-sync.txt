# Content Automation Milestone: Weekly Check

Tue, 14 Apr 26 · claudia.tita@tourlane.com, kiryl.karetnikau@tourlane.com, pedro.alves@tourlane.com, PR-20 HQ-2-China (8) [Camera, Google Meet, Mic, TV, Tablet]

### Veto Migration Progress

- Nearly complete endpoint specifications for catalog replacement
  - Fast-track vibration steps identified
  - Search endpoint development needed in catalog
  - Accommodation fetch endpoint improvements made
  - Better understanding of price refresh functionality
- Migration steps outlined as next priority
  - Concrete task definition pending
  - All missing capabilities for accommodations will be addressed
- Destinations more advanced than expected

### Expedia Certification & Launch Strategy

- Expedia certification status
  - TNCs updates in progress for front-end and Germany.com
  - Additional TNC changes required, completion expected end of day
  - Test booking in staging remains open item
    - Must trigger booking from trip planning
    - Must show booking in booking platform
- Two-phase launch approach planned
  - Similar to hotel rollout model
  - Allows feedback collection and rollback capability
  - Prevents mass content changes with unknown conversion impact

### Resource Allocation & Prioritization Decisions

- Current workload assessment
  - Veto migration: Priority #1 (all hands)
  - API outsourcing: Ongoing but shouldn’t block team significantly
    - Expected: 1 engineer for onboarding support
    - Goal: Minimal team impact once gateway ready
- Major decision points requiring resolution
  - Catalog capabilities vs. supply domain investment
  - Room level mapping API for smart booking needs entire supply domain migration
  - Legacy system replacement strategy (everything except catalog considered legacy)
- Destination launches blocked
  - Finland and Sweden work can continue (already started)
  - No new destination launches planned for rest of year
  - Formal sign-off needed from Karen for 10-month block
  - RAPID document to be drafted for decision formalization
- Resource allocation rule: Minimum 2 people per topic
  - Current constraint: 5 engineers total
  - Maximum 3 concurrent topics sustainable
